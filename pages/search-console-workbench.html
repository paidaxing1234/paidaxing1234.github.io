Revenue leak radar

把 Search Console 变成收入泄漏地图

Google 官方已经把 AI features 的曝光纳入 Search Console 的 Performance report，AI Overviews 和 AI Mode 也没有额外特殊优化要求。这个工具把报表贴进来后，直接拆成标题、内容、AI 入口和广告收益四类修复动作。

AdSense 广告位复盘工作台顶部广告
官方边界：AI features 仍然遵循基础 SEO，重点还是可索引、可抓取、内容有帮助。Search Console 的 Performance report 能看查询、页面和国家趋势，也能作为 AI features 相关曝光的监测入口。这个页面做的是把导出数据变成修复建议，不是猜测排名。

运营工作台

粘贴 Search Console / AdSense CSV

支持列：cluster, impressions, clicks, dwell, pages, rpm, ai_impressions, ai_clicks, density。你可以直接粘贴导出的报表行。

收入泄漏指数 0

把报表贴进来后开始分析。

动作 1

有展示没点击

优先改标题、摘要和首屏答案块，让搜索结果更像用户要的答案，而不是宽泛介绍。

动作 2

有点击没停留

补公式、表格、示例、FAQ 和真实案例。说明用户点进来了，但页面还不够解决问题。

动作 3

AI 入口吃掉点击

把内容写成非 commodity 的答案页，增加原创数据、流程、截图和对比，让 AI features 更容易带来支持链接。

动作 4

广告收益开始出现

把高 RPM 页面做成站点主骨架，扩展相同意图的细分词簇，形成稳定的内链和收入池。

信号说明建议动作
展示高 / 点击低标题和摘要不够对口，或竞争结果更强重写标题，前 120 字先给短答案
点击高 / 停留短用户点进来后没找到价值补表格、FAQ、例子、公式和对比
AI 入口高 / Web CTR 低页面有被 AI features 触达，但没有转成点击增加非 commodity 内容，给出更强的原创价值
页面多 / 收益低页面群质量不够高，或者词不够商业删薄页，切到更强商业意图的词簇
RPM 高 / 展示少方向对，但规模太小围绕同类词扩 5 到 10 个相关页

优先级

最该先改的页面簇

按泄漏指数、收入回收空间、AI 压力和页面簇成熟度排序。

优先处理
最值钱的复盘

不是“我又更新了”，而是“我知道该改哪一页”

如果没有这个动作闭环，站点很容易变成一堆页面堆积。真正能跑起来的内容站，是持续用数据淘汰薄页、强化高展示词簇。

真实门槛

没有 Search Console，就没有增长地图

上线前都是假设。只有当真实域名、真实数据和真实用户进来后，才知道哪个细分词簇值得继续投入。

真实判断：这个工作台不代表你已经有数据，它只是把上线后的复盘方法提前做出来。要真正用起来，还是得先绑定真实域名并接入 Search Console。
